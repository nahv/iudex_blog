## Metadata de Cada Post

Cada artículo en el HTML debe tener esta estructura de metadatos (para filtrado y SEO):

### HTML Structure

```html
<article class="blog-post" data-category="automatizacion">
  <header class="blog-post__header">
    <span class="blog-post__badge" data-category="automatizacion">Automatización</span>
    <h1>Titulo del Artículo</h1>
    <div class="blog-post__meta">
      <span class="blog-post__author">Por Nahuel</span>
      <span class="blog-post__date">8 de abril, 2026</span>
      <span class="blog-post__reading-time">7 minutos de lectura</span>
    </div>
  </header>
  <main class="blog-post__content">
    <!-- Contenido aquí -->
  </main>
</article>
```

### En blog/index.html (card del post)

```html
<div class="blog-card" data-category="automatizacion">
  <img src="..." alt="..." class="blog-card__image">
  <div class="blog-card__content">
    <span class="blog-card__badge">Automatización</span>
    <h2 class="blog-card__title">Título del Artículo</h2>
    <p class="blog-card__excerpt">Las primeras 150 caracteres del artículo...</p>
    <div class="blog-card__meta">
      <span class="blog-card__author">Por Nahuel</span>
      <span class="blog-card__date">8 de abril</span>
      <span class="blog-card__reading-time">7 min</span>
    </div>
    <a href="/blog/url.html" class="blog-card__link">Leer más →</a>
  </div>
</div>
```

### Variables Clave

| Variable | Ejemplo | Nota |
|----------|---------|------|
| `data-category` | "automatizacion" | Minúsculas, guión (no espacio) |
| `data-category` badge | "Automatización" | Mayúscula inicial para UI |
| Author | "Nahuel" o "Maxi Bury" | Nombre real |
| Date | "8 de abril, 2026" | Formato: "d de mes, YYYY" |
| Reading Time | "7 minutos" | Calcular: ~200 palabras/minuto |

## Estructura de un Artículo

Cada post de blog debe seguir esta estructura para máxima legibilidad y SEO:

### 1. Hero Section
```html
<header class="article-hero">
  <span class="article-hero__badge">Automatización</span>
  <h1 class="article-hero__title">Título del Artículo</h1>
  <div class="article-hero__meta">
    <img src="author.jpg" alt="Nahuel" class="article-hero__author-photo">
    <div>
      <p class="article-hero__author">Por <strong>Nahuel</strong></p>
      <p class="article-hero__date-time">8 de abril de 2026 · 7 minutos</p>
    </div>
  </div>
</header>
```

**Estilos**:
- Badge: fondo gold, texto ink, pequeño (12px)
- Título: Playfair Display, 48px (desktop), 32px (mobile)
- Meta: DM Sans, 14px, gris oscuro
- Separador visual: línea delgada (1px cream-dark) después de meta

### 2. Contenido Estructurado

```html
<article class="article-content">
  <p>Párrafo introductorio (set context)...</p>

  <h2>Sección 1</h2>
  <p>Contenido...</p>
  <ul>
    <li>Punto 1</li>
    <li>Punto 2</li>
  </ul>

  <h3>Subsección</h3>
  <p>Contenido...</p>

  <blockquote class="article-quote">
    <p>"Cita relevante o insight clave."</p>
    <cite>— Fuente o autor</cite>
  </blockquote>

  <h2>Sección 2</h2>
  <!-- ... -->
</article>
```

**Recomendaciones**:
- Max 3 niveles de headings (h1, h2, h3)
- Párrafos cortos (3-4 líneas max)
- Listas para puntos
- Blockquotes para citas/insights
- Imágenes cada 3-4 párrafos (con alt text descriptivo)

### 3. Sección CTA al Final

```html
<aside class="article-cta">
  <div class="article-cta__box">
    <h3>¿Interesado en optimizar tu práctica legal?</h3>
    <p>Iudex es software moderno diseñado para abogados argentinos. Organiza expedientes, automatiza documentos, funciona sin internet.</p>
    <a href="/contacto" class="button button--primary">Solicitar acceso anticipado</a>
  </div>
</aside>
```

**Styling**:
- Fondo cream-dark, padding var(--space-lg)
- Borde rounded (16px)
- Contenido centrado

### 4. Meta Tags en <head>

```html
<title>Nombre del Artículo - Iudex Blog</title>
<meta name="description" content="Breve resumen (160 chars max)">
<meta name="keywords" content="palabra1, palabra2, palabra3">
<meta name="author" content="Nahuel">
<meta property="og:title" content="Nombre del Artículo">
<meta property="og:description" content="Resumen para redes">
<meta property="og:image" content="/assets/images/blog-thumb.jpg">
<link rel="canonical" href="https://iudex.com.ar/blog/url.html">
```

## Convenciones de Contenido

### Audiencia
- Abogados argentinos (principal)
- Graduados recientes, paralegales (secundaria)
- Docentes de derecho (tertiary)

### Tono
- Profesional pero accesible (sin jargon innecesario)
- Conversacional (voseo rioplatense)
- Educativo (enseña, no vende)
- Honesto (evita hype, no oversells Iudex)

### Idioma
- Español argentino preferido (pero neutral también aceptado)
- Voseo: "vos necesitás", no "tú necesitas"
- Términos locales: "juzgado", "escribano", "expediente" (no traducir)

### Longitud
- Posts cortos: 4-5 minutos (~900 palabras)
- Posts medianos: 7-8 minutos (~1400 palabras)
- Posts largos: 10+ minutos (~2000+ palabras)
- Meta: mantener bajo 10 min (si es más, dividir en serie)

### Estructura de Contenido
- Hook (primeros 2 párrafos): capta atención, define tema
- Cuerpo (mitad): desarrolla idea con ejemplos, datos, evidencia
- Solución/cierre (final): qué hacer al respecto, CTA

### SEO Local
- Mencionar provincias, ciudades, tribunales argentinos (cuando relevante)
- Referencias a leyes argentinas (Código Civil, Código Procesal)
- Keywords: "abogado argentina", "software legal", "gestion expedientes"

### Qué Evitar
- Fake statistics ("90% de abogados...")
- Comparativas directas a competencia
- Demasiadas CTAs (máximo 1-2 explícitas)
- Diccionarios/listas que podrían ser Wikipedia
- Contenido obsoleto sin fecha de actualización

## Checklist al Agregar Nuevo Post

Cuando se escribe y publica un nuevo artículo, verificar:

1. **Contenido**
   - [ ] Hook fuerte (primeros 100 chars captan atención)
   - [ ] Mínimo 800 palabras
   - [ ] Máximo 3000 palabras (o dividir en serie)
   - [ ] Jerarquía clara (h1, h2, h3 bien distribuidos)
   - [ ] Al menos una lista o blockquote
   - [ ] Al menos 1-2 imágenes relevantes

2. **Metadata**
   - [ ] Title: <60 chars, contiene keyword
   - [ ] Description: <160 chars, call-to-action implícito
   - [ ] Keywords: 5-10, incluye variaciones (singular/plural)
   - [ ] Author: nombre correcto
   - [ ] Date: formato "DD de mes, YYYY"
   - [ ] Reading time: cálculo correcto (words/200)
   - [ ] Category: uno de los 6 definidos
   - [ ] Featured: sí/no
   - [ ] Canonical: URL correcta

3. **HTML**
   - [ ] Semántica: `<article>`, `<header>`, `<main>`
   - [ ] Headings jerárquicos (sin saltar h1 → h3)
   - [ ] Alt text en todas las imágenes
   - [ ] Links internos (mínimo 1 link a otro post)
   - [ ] Links externos (si aplica, en tab nuevo)

4. **Imágenes**
   - [ ] Tamaño: 1200x600px mínimo (featured image)
   - [ ] Formato: JPEG o WebP
   - [ ] Tamaño archivo: <100KB
   - [ ] Nombres descriptivos: "gestion-expedientes-hero.jpg"
   - [ ] Alt text: descriptivo, no "imagen1"

5. **Publicación**
   - [ ] Actualizar `blog/index.html` (agregar card)
   - [ ] Crear archivo `/blog/url-del-articulo.html`
   - [ ] Testear links internos
   - [ ] Revisar en mobile (375px, 768px)
   - [ ] Lighthouse score >90

6. **Promoción**
   - [ ] Preparar imagen para redes (Instragram, LinkedIn)
   - [ ] Draft de email newsletter (si aplica)
   - [ ] Hashtags preparados
   - [ ] CTA claro en redes
